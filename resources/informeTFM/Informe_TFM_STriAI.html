1. Resumen / Abstract

Resumen

El presente Trabajo de Fin de Máster aborda el desarrollo de un sistema de triaje multimodal basado en inteligencia artificial para la clasificación de pacientes en servicios de urgencias médicas en Colombia, conforme a la Resolución 5596/2015. El sistema, denominado STriAI, integra datos clínicos estructurados con procesamiento de lenguaje natural (NLP) sobre el motivo de consulta en texto libre, implementando una arquitectura de fusión temprana (Early Fusion) que combina ambas modalidades mediante XGBoost.

El pipeline de entrenamiento procesó 176,641 registros de cuatro fuentes colombianas. Se entrenaron cinco arquitecturas: LR, RF, XGBoost unimodal, Early Fusion y Late Fusion. El modelo ganador alcanzó un F1 Macro de 0.1895 y una exactitud del 79.86%. La aplicación demo se implementó sobre Streamlit con 14 pantallas, SQLite, RBAC de 5 roles y auditoría inmutable.

Palabras clave: triaje hospitalario, inteligencia artificial, machine learning, NLP, XGBoost, BERT, sistema de apoyo a la decisión clínica, Colombia.

Abstract

This Master's Thesis presents a multimodal AI-based triage system for emergency department patient classification in Colombia (Resolution 5596/2015). The system integrates structured clinical data with NLP on free-text chief complaints via an Early Fusion XGBoost architecture. Five architectures were trained on 133,047 valid cases from Colombian healthcare sources. The winning model achieved Macro F1 = 0.1895 and accuracy = 79.86%. A Streamlit demo application with 14 screens, RBAC, and immutable audit was implemented.

Keywords: hospital triage, artificial intelligence, machine learning, NLP, XGBoost, BERT, clinical decision support, Colombia.

9. Conclusiones y Trabajo Futuro

9.1 Conclusiones Principales

  1. Viabilidad técnica demostrada: Sistema completo de triaje multimodal funcional con pipeline reproducible y demo interactiva.
  2. Brecha de rendimiento: F1=0.19 vs meta 0.82. Causa principal: escasez de features clínicas (3 vs decenas en benchmarks).
  3. Late Fusion clínicamente superior: Único modelo que detecta las 5 clases (incluyendo críticos I-II). La métrica F1 Macro no captura utilidad clínica.
  4. Limitación de datos colombianos: Los datasets abiertos solo contienen datos administrativos.
  5. Documentación exhaustiva: Manuales de usuario, técnico, modelos, instalación y arquitectura generados.

9.3 Lecciones Aprendidas

  1. La calidad de los datos es el techo del modelo — ninguna arquitectura sofisticada compensa la ausencia de features clínicas.
  2. F1 Macro no es adecuada para triaje — trata igual un error en Nivel I (muerte prevenible) que en Nivel V (consulta externa).
  3. El desbalance de clases en datos reales es la norma — diseñar asumiéndolo desde el inicio.
  4. Streamlit + SQLite + joblib fue acertado para prototipo, pero limita escalabilidad a producción.
