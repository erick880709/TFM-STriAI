1. Resumen Ejecutivo

El pipeline de entrenamiento del Sistema de Triaje Multimodal IA (STriAI) entrena y evalúa 5 arquitecturas de modelos sobre datos reales de urgencias hospitalarias en Colombia, combinando features estructuradas (edad, signos vitales derivados) con embeddings NLP generados por un modelo BERT multilingüe a partir del texto libre del motivo de consulta.

MétricaValor
Modelo ganadorEarly Fusion (XGBoost + BERT)
F1 Macro0.1895
Accuracy79.86%
Features totales387 (3 estructuradas + 384 NLP)
Filas de entrenamiento133,047
Clases objetivoI, II, III, IV, V (Res. 5596/2015)
Tiempo de ejecución~17 minutos (CPU)
⚠️ Nota importante: El rendimiento del modelo está limitado por la calidad y cantidad de features disponibles en los datasets colombianos. Los datasets reales utilizados carecen de signos vitales completos y comorbilidades. El F1 Macro de 0.19 está muy por debajo de la meta de 0.82. Se recomienda enriquecer los datasets con fuentes clínicas más completas.

2.3 Limitaciones de los Datasets

  • ❌ Sin signos vitales: No hay frecuencia cardíaca, presión arterial, saturación O₂, temperatura, frecuencia respiratoria
  • ❌ Sin comorbilidades: No hay datos de diabetes, hipertensión, cáncer, etc.
  • ❌ Sin escalas clínicas: No hay Glasgow, NEWS2, MEWS, escala de dolor
  • ✅ Con texto diagnóstico: El dataset de San Juan de Dios sí tiene diagnóstico textual (código CIE-10 + descripción)
  • ✅ Con nivel de triaje: 133,047 de 176,641 filas tienen nivel de triaje válido

4.2 Análisis por Modelo

Regresión Logística (LR)

  • Fortaleza: Mejor Recall en Nivel II (81.84%)
  • Debilidad: F1 Macro = 0.01 — predice casi todo como Nivel II
  • Conclusión: Inútil como clasificador general, sufre por el desbalance extremo

Random Forest (RF)

  • Fortaleza: Recall balanceado entre niveles II-V
  • Debilidad: F1 Macro muy bajo (0.03), no detecta Nivel I

XGBoost (Unimodal)

  • Fortaleza: Recall Nivel III = 100%
  • Debilidad: Ignora completamente Niveles I, II, IV, V
  • Conclusión: Aprendió a predecir siempre la clase mayoritaria

Early Fusion 🥇

  • Fortaleza: Mejor F1 Macro global, Recall III 99.3%
  • Debilidad: Recall I = 0%, V = 0% — no detecta los extremos
  • Arquitectura: XGBoost sobre concatenación estructurado + NLP (387 dims)

Late Fusion

  • Fortaleza: Único modelo que detecta los 5 niveles. Mejor Recall I (20%), IV (22.7%), V (14.3%)
  • Debilidad: F1 Macro bajo (0.10)
  • Arquitectura: Stacking: XGBoost(struct) + LR(NLP) → LR meta-modelo
💡 Recomendación clínica: Late Fusion es más útil para tamizaje (detecta pacientes críticos I-II), mientras que Early Fusion es mejor para clasificación poblacional. Para uso real, considerar Late Fusion con ajuste de métrica.

7. Explicabilidad (SHAP / Feature Importance)

7.1 Limitación Técnica

La versión instalada de SHAP (0.51.0) es incompatible con XGBoost 3.2.0. Como fallback, se usan las feature_importances_ nativas de XGBoost (ganancia de información por split).

9. Cómo Re-ejecutar el Pipeline

9.1 Requisitos

  • Python 3.11+ con entorno virtual
  • pip install -r sistema-triaje-ia/requirements.txt
  • Datasets CSV en datasets/
  • ~2 GB espacio en disco (modelo NLP ~420 MB)
  • 8 GB RAM mínimo

9.2 Comando

# Desde TFM-FINAL/
python run_pipeline.py

# Con opciones:
python run_pipeline.py \
  --datasets-dir ./datasets \
  --output-dir ./models \
  --nlp-model multilingual

10. Limitaciones y Recomendaciones

10.1 Limitaciones Actuales

  1. Dataset empobrecido: Sin signos vitales, comorbilidades, ni escalas clínicas
  2. Desbalance extremo: 88.5% Nivel III
  3. AUC-ROC = 0.00: Posible bug en sklearn 1.9
  4. SHAP no funcional: Incompatibilidad de versiones
  5. Modelo NLP genérico: MiniLM no especializado en dominio médico
  6. Features derivadas sintéticas: pam, shock_index basados en valores imputados

10.2 Recomendaciones para Mejora
