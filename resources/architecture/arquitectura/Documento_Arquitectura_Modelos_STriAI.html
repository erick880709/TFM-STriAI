1. Visión General

1.1 Objetivo del Sistema ML

Predecir el nivel de triaje (I-V) de un paciente que ingresa a urgencias, utilizando dos modalidades de datos:

  • Modalidad estructurada: Signos vitales, datos demográficos, comorbilidades → 3 features tras encoding
  • Modalidad textual: Motivo de consulta en texto libre → 384 dimensiones tras embedding BERT

💡 Ironía arquitectónica: Late Fusion es el único modelo que detecta las 5 clases (incluyendo pacientes críticos Nivel I-II), pero "pierde" en F1 Macro porque sacrifica Nivel III para ganar en las clases minoritarias. Se recomienda cambiar la métrica de selección a F1 ponderado por criticidad clínica.

6. Arquitectura de Inferencia

6.1 Flujo de Inferencia

  1. Verificar modelo: Si no está cargado, leer models/active_version.txt → cargar .joblib
  2. Construir feature vector: Leer datos de SQLite (Paciente + SignosVitales + EvaluacionClinica) → mapear a features
  3. Preprocesar: scaler.transform() + encoder.transform()
  4. NLP: SentenceTransformer.encode(motivo_texto) → vector (384,)
  5. Concatenar: hstack([X_struct, X_nlp]) → (387,)
  6. Predecir: model.predict_proba(X) → 5 probabilidades
  7. Aplicar umbrales: _apply_thresholds(probas) → clase final
  8. Persistir: INSERT INTO PrediccionIA + UPDATE EventoTriaje

6.2 Patrón Singleton

_inference_service = None  # Variable global de módulo

def get_inference_service(models_dir=None):
    global _inference_service
    if _inference_service is None:
        _inference_service = InferenceService(models_dir)
        _inference_service.load_model()
    return _inference_service
